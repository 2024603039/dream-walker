# Dream Walker

## 1. 프로젝트 개요

Dream Walker는 짧은 1인칭 공포 게임이다.

이 게임은 내가 실제로 어렸을 때 꾸었던 꿈에서 아이디어를 얻었다.

어느 날 꿈속에서 엄마를 만났는데, 나는 그것이 꿈이라는 사실을 인지하고 있었다. 그래서 꿈속의 엄마에게 "이건 꿈이야."라고 말했는데, 그 순간 엄마의 표정이 바뀌면서 나를 미친 듯이 쫓아오기 시작했다.

나는 비명을 지르며 잠에서 깼고, 현실에서도 엄마가 놀라서 나에게 달려오고 있었다. 꿈속에서 보았던 장면과 현실의 장면이 겹쳐 보이면서 마치 꿈이 끝나지 않은 것 같은 느낌을 받았다.

가장 익숙하고 편안해야 할 존재가 갑자기 공포의 대상으로 변하는 경험이 굉장히 강하게 기억에 남았다. 그래서 이 경험을 직접 게임으로 만들어 보고 싶다는 생각이 들었고, Dream Walker를 기획하게 되었다.

---

## 2. 게임 진행 방식

### 2-1. 시작

게임은 침대에서 시작된다.

플레이어는 어두운 방 안에서 눈을 뜨고, 방 안에는 엄마 NPC가 서 있다.

플레이어는 WASD와 마우스를 이용하여 자유롭게 움직이고 시점을 변화시킬 수 있다.

![시작 화면](./start-room.png)

---

### 2-2. 엄마와의 대화

엄마에게 가까이 다가가 E 키를 누르면 대화를 할 수 있다.

엄마는 "일어났니? 학교 가야지."라고 말하며 두 개의 선택지를 제시한다.

* [1] 이건 꿈이다
* [2] 현실이다

이 상황에 이상함을 느끼면 [1]을 누르고, 현실이라고 판단하면 [2]를 누르면 된다.

![엄마와 대화](./mom-dialog.png)

---

### 2-3. 악몽의 시작

플레이어가 "이건 꿈이다"를 선택하면 게임이 본격적으로 시작된다.

방 전체가 붉은색으로 물들고 조명이 변경된다.

엄마는 더 이상 평범한 모습이 아니라 플레이어를 따라다니며 꿈에서 깨지 못하도록 위협하는 존재로 바뀐다.

![붉은 방](./red-room.png)

---

### 2-4. 기억 조각 수집

플레이어는 엄마를 피해 다니며 방 안에 흩어져 있는 기억 조각 4개를 찾아야 한다.

기억 조각은 검은 오브젝트 형태로 설정했다. 

방의 각 꼭짓점에 위치하도록 설계했다.

플레이어가 가까이 다가간 뒤 E 키를 누르면 수집된다.

![기억 조각](./memory-fragment.png)

---

기억 조각을 수집하면 현재 수집 개수가 화면에 표시된다.

플레이어는 4개의 기억 조각을 모두 찾아야 한다.

기억 조각을 모두 수집해야 꿈에서 깰 수 있다.

![기억 개수 표시](./memory-count.png)

---

### 2-5. 꿈에서 깨어남

기억 조각을 모두 수집하면 화면이 하얗게 변한다.

방이 하얗게 변한 뒤 플레이어는 시작과 똑같이 침대에서 눈을 뜬다.

![화이트 스크린](./white-screen.png)

---

이후 방은 원래 상태로 복구된다.

붉은 조명과 공포 분위기가 사라지고 처음 게임을 시작했을 때의 공간으로 돌아온다.

두려움으로 얼른 엄마를 찾으면 엄마가 꿈에서와 똑같은 위치로 서있다.

'이것 또한 꿈인가' 라는 의문이 생기도록 엄마의 위치를 게임 시작과 같은 위치로 수미상관이다.

![깨어난 방](./wake-room.png)

---

### 2-6. 마지막 장면

플레이어는 다시 엄마에게 다가간다.

엄마는 처음에 꿈에서 했던 말과 같은 대사를 친다. 

하지만 곧 플레이어에게 다시 가까이 다가오며 플레이어가 이 공포의 꿈이 아직 끝나지 않았음을 느낀다. 

![마지막 엄마 등장](./ending-mom.png)

---

### 2-7. 엔딩

게임은 "깨어난 줄 알았다."라는 문장과 함께 종료된다.

꿈에서 깨어났다고 생각했지만 사실은 끝나지 않았다는 뜻이다.

이는 내가 진짜 피곤할 때 꿨던 꿈 중에서 분명 깼는데 그것 또한 꿈이었고 마지막에는 현실에서도 이게 꿈인지 의심했던 경험에서 모티브를 삼았다. 

![엔딩 화면](./end-screen.png)

---

## 3. 강의 내용과 구현 내용 매핑

### 3-1. First Person Camera

수업 시간에 배운 1인칭 카메라 방식을 사용하였다.

마우스를 움직여 시점을 회전할 수 있고, WASD 키를 이용해 직접 공간을 탐험할 수 있다.

공포 게임 특성상 플레이어가 직접 공간 안에 들어와 있는 느낌이 중요하다고 생각하여 1인칭 시점을 선택하였다.

3인칭 시점을 선택하지 않은 이유는 공포 게임 특성 상 주변시야의 한계가 있어야 더 무섭기 때문이다. 

위를 보고 있으면 갑자기 옆에서 다가오고, 오른쪽으로 보면 왼쪽에서 다가오는것이 더 무섭다고 판단했다.

![게임 플레이 화면](./start-room.png)

---

### 3-2. Scene Graph

Three.js의 Scene 구조를 사용하여 침대, 벽, 문, 엄마 NPC, 기억 조각 등을 배치하였다.

프로젝트를 진행하면서 Scene Graph가 생각보다 중요하다는 것을 느꼈다.

특히 꿈에서 깨어나는 연출을 만들 때 Scene 내부 오브젝트들을 순회하며 색상과 조명 값을 원래 상태로 복구하는 과정에서 많이 사용하였다.

![Scene Graph 적용 장면](./wake-room.png)

---

### 3-3. Material

수업 시간에 Material이 물체의 외형을 결정한다는 내용을 배웠다.

이번 프로젝트에서는 같은 공간이라도 Material 색상을 바꾸어 전혀 다른 분위기를 만들 수 있었다.

평범한 방은 차가운 색을 사용하였고, 악몽 상태에서는 붉은 계열 색상으로 변경하였다.

![Material 변경 장면](./red-room.png)

---

### 3-4. Lighting

조명은 게임 분위기 표현에 가장 큰 영향을 주는 요소였다.

처음에는 단순히 색상만 바꾸려고 했지만 생각보다 무섭지 않았다.

그래서 조명 색상까지 함께 변경하니 공간 전체 분위기가 완전히 달라졌다.

![Lighting 적용 장면](./red-room.png)

사용 코드

```javascript
const redLight = new THREE.PointLight(0xff0000, 6);
scene.add(redLight);
```

붉은 조명을 추가하여 악몽 공간의 분위기를 표현하였다.

처음에는 Material 색상만 변경했지만 공포 분위기가 충분하지 않았기 때문에 조명 색상도 함께 변경하였다.
---

### 3-5. Point Light

기억 조각 주변에는 Point Light를 배치하였다.

아무 표시가 없으면 기억 조각을 찾기 어렵다고 판단하여 작은 빛이 퍼지도록 만들었다.

플레이어는 자연스럽게 빛을 따라 기억 조각을 발견할 수 있다.

![Point Light 적용](./memory-fragment.png)

---

### 3-6. Emissive Material

문과 일부 오브젝트에는 Emissive 값을 사용하였다.

일반적인 색상만 사용하는 것보다 빛이 나는 것처럼 보이기 때문에 플레이어가 목표 지점을 쉽게 확인할 수 있었다.

![Emissive 적용 장면](./red-room.png)

---

### 3-7. Animation

엄마 NPC는 단순히 이동만 하는 것이 아니라 걸어오는 것처럼 보이도록 애니메이션을 적용하였다.

몸이 위아래로 움직이도록 하여 정적인 움직임보다 자연스럽게 표현하였다.

특히 추격 구간에서 엄마가 계속 플레이어를 향해 다가오도록 구현하여 긴장감을 높이고자 하였다.

![엄마 추격 장면](./chase.png)

사용 코드

```javascript
momNPC.userData.walkPhase += 0.18;

momNPC.position.y =
Math.abs(Math.sin(momNPC.userData.walkPhase)) * 0.04;
```

NPC가 단순히 미끄러지듯 이동하는 것이 아니라 실제로 걸어오는 것처럼 보이도록 구현하였다.

사인 함수를 사용하여 몸이 위아래로 움직이도록 만들었다.
---

### 3-8. Collision 및 이동 제한

플레이어가 맵 밖으로 나가는 문제를 막기 위해 이동 범위를 제한하였다.

카메라 위치를 특정 범위 안으로 고정하여 의도하지 않은 공간으로 이동하지 못하도록 만들었다.

게임을 플레이하면서 벽을 뚫고 나가거나 이벤트 공간을 벗어나는 문제가 발생하지 않도록 구현하였다.

![이동 제한이 적용된 장면](./start-room.png)

---

### 3-9. Interaction System

게임의 주요 이벤트는 모두 상호작용 시스템을 통해 진행된다.

엄마와의 대화, 기억 조각 수집 등이 모두 E 키 상호작용을 사용한다.

처음에는 가까이 가기만 해도 이벤트가 실행되었지만 의도하지 않은 상황이 자주 발생하여 상호작용 키를 추가하였다.

덕분에 플레이어가 원하는 순간에 이벤트를 진행할 수 있게 되었다.

![상호작용 장면](./mom-dialog.png)

사용 코드

```javascript
if (
 camera.position.distanceTo(momNPC.position) < 2.5
) {
 statusText.textContent =
 '엄마: 일어났니? 학교 가야지.';
}
```

플레이어가 NPC 근처에 접근했을 때만 상호작용이 가능하도록 구현하였다.

이를 통해 의도하지 않은 이벤트 발생을 줄일 수 있었다.
---

### 3-10. Distance Check

엄마와 대화하거나 기억 조각을 획득할 때 거리 계산을 사용하였다.

플레이어가 충분히 가까이 왔을 때만 이벤트가 발생하도록 구현하였다.

실제 게임을 만들면서 거리 계산이 생각보다 자주 사용된다는 것을 알게 되었다.

특히 대화 시스템과 기억 조각 수집 시스템 모두 거리 계산을 기반으로 동작한다.

![거리 계산이 사용된 장면](./memory-fragment.png)

---

### 3-11. State 관리

게임은 여러 단계로 구성되어 있다.

1. 시작 상태
2. 꿈을 인지한 상태
3. 기억 조각 수집 상태
4. 꿈에서 깨어난 상태
5. 엔딩 상태

처음에는 이벤트들이 서로 꼬이는 문제가 많았다.

그래서 현재 상태를 저장하는 변수를 만들어 각 단계별로 다른 이벤트가 실행되도록 수정하였다.

이 부분이 게임 전체 구조를 만드는 데 가장 중요한 역할을 하였다.

![상태 변화 예시](./memory-count.png)

사용 코드

```javascript
if (memories >= 4) {
 triggerWakeUp();
}
```

기억 조각을 모두 수집했을 때 다음 상태로 넘어가도록 구현하였다.

게임 전체를 상태(State) 기반으로 관리하여 이벤트가 서로 충돌하지 않도록 구성하였다.
---

### 3-12. Surfel Global Illumination

수업 시간에 DDGI(Dynamic Diffuse Global Illumination)와 Surfel 기반 간접광 표현 기법에 대해 학습하였다.

GI(Global Illumination)는 빛이 물체 표면에 반사된 뒤 다른 표면에도 영향을 주는 현상을 의미한다. 일반적인 Direct Lighting보다 훨씬 현실적인 조명 표현이 가능하지만 계산량이 매우 크다는 특징이 있다.

이번 프로젝트에서는 실제 DDGI 수준의 간접광 계산을 구현하지는 않았지만, 강의에서 배운 Surfel(Surface Element) 개념을 게임 내에서 시각적으로 확인할 수 있도록 구현하였다.

G 키를 누르면 Surfel Point Visualization이 활성화된다.

파란색 점은 일반적인 Surfel을 의미하며, 노란색 점은 상대적으로 더 강한 빛 정보를 가진 Surfel을 표현하였다.

이를 통해 강의 시간에 학습한 GI와 Surfel 개념을 단순 이론으로 끝내지 않고 실제 프로젝트 안에서 확인할 수 있도록 구성하였다.

![Surfel OFF](./surfel-off.png)

일반 렌더링 상태

![Surfel ON](./surfel-on.png)

Surfel Visualization 활성화 상태

사용 코드

```javascript
if (event.key.toLowerCase() === 'g') {
 surfelVisible = !surfelVisible;
}
```

G 키를 눌렀을 때 Surfel Visualization이 켜지고 꺼지도록 구현하였다.

강의에서 학습한 Surfel 개념을 게임 안에서 직접 확인할 수 있도록 만들었다.

## 4. 구현 과정에서의 시행착오

### 4-1. 엄마 NPC 추격 문제

처음에는 엄마가 플레이어를 제대로 바라보지 않고 이동하였다.

그래서 플레이어 위치를 기준으로 NPC가 항상 바라보도록 수정하였다.

수정 이후에는 실제로 플레이어를 추격하는 느낌이 훨씬 강해졌다.

---

### 4-2. 꿈에서 깬 뒤에도 붉은 방이 유지되는 문제

처음 구현했을 때는 꿈에서 깨어난 뒤에도 붉은 조명과 색상이 그대로 남아 있었다.

그래서 꿈에서 깨어나는 함수에서 Scene 내부의 Material과 Light 값을 원래 상태로 되돌리도록 수정하였다.

이 과정을 통해 현실로 돌아온 느낌을 표현할 수 있었다.

---

### 4-3. 엔딩 연출 문제

마지막 점프 스케어 장면에서 엄마의 얼굴이 아니라 몸통이나 뒷모습이 확대되는 문제가 발생하였다.

카메라 방향과 NPC 회전을 수정하여 항상 얼굴이 플레이어를 향하도록 변경하였다.

수정 후에는 의도했던 공포 연출이 정상적으로 동작하였다.

---

### 4-4. 함수 중복 선언 문제

개발 과정에서 interact 함수와 변수들이 중복 선언되어 게임이 실행되지 않는 문제가 발생하였다.

브라우저 콘솔을 통해 오류 위치를 확인한 뒤 중복된 선언을 제거하였다.

이번 프로젝트를 진행하면서 콘솔 오류를 읽는 능력이 생각보다 중요하다는 것을 느꼈다.

---

### 4-5. GitHub Pages 배포 문제

게임을 GitHub Pages로 배포하는 과정에서 정적 파일 경로 문제가 발생하였다.

로컬 환경에서는 정상적으로 실행되었지만 GitHub Pages에서는 JavaScript 파일과 모델 파일을 찾지 못해 게임이 실행되지 않았다.

빌드 결과물(dist) 구조를 확인하고 경로를 수정하여 문제를 해결하였다.

또한 GitHub Pages가 기본적으로 README를 먼저 표시하는 문제도 있었기 때문에 index.html 경로를 수정하여 게임이 정상적으로 실행되도록 변경하였다.

이번 프로젝트를 통해 구현뿐만 아니라 실제 배포 과정도 중요하다는 점을 알 수 있었다.

## 5. 느낀 점

이번 프로젝트를 진행하면서 수업 시간에 배운 내용을 단순히 따라 하는 것과 실제 게임에 적용하는 것은 많이 다르다는 것을 느꼈다.

처음에는 단순한 공포 게임을 만들 생각이었지만 구현을 진행하면서 카메라, 조명, 상호작용, 애니메이션, 상태 관리 등 다양한 요소들이 함께 동작해야 한다는 것을 알게 되었다.

특히 작은 버그 하나 때문에 게임 전체가 실행되지 않는 경우도 많았기 때문에 디버깅의 중요성을 많이 느꼈다.

또한 내가 실제로 경험했던 꿈을 게임으로 표현할 수 있어서 재미있었고, 평소에는 무섭게만 느꼈던 기억을 직접 작품으로 만들어 본 점이 가장 인상 깊었다.

강의에서 배운 내용을 실제 프로젝트에 적용해 볼 수 있었던 좋은 경험이었다.
